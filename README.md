# Low-Latency ITCH Order Book — C++ and RTL

A limit-order-book pipeline implemented twice: an optimized C++17 engine and a
synthesizable SystemVerilog engine. Both consume the same normalized NASDAQ
ITCH 5.0 event stream (`A F E C X D U P S`). The verification harness compares
the RTL book with the C++ reference after every committed update.

The input format uses standard big-endian ITCH fields and a two-byte message
length, with timestamps normalized to eight bytes. The converter for native
NASDAQ recordings is in `gui/data/nasdaq_itch.py`.

## Quick start

```bash
bash run_all.sh   # data, software tests/benchmarks, RTL/board tests, analysis
make test         # C++ tests only
make sim          # cycle-accurate RTL tests
make sim-board    # UART bridge + FPGA configuration in Verilator
```

Requirements and Windows/MSYS2 setup are in
[docs/BUILDING.md](docs/BUILDING.md). Synthetic data is deterministic, so the
default verification flow does not require a market-data download.

## Verified results

These numbers were regenerated by `run_all.sh` on the machine recorded in
[docs/results.json](docs/results.json). Software timing uses serialized TSC
samples, subtracts timer overhead, pins the process when supported, and drops
samples that migrate between logical processors.

| Measurement | Result |
|---|---:|
| C++ parse + update latency | 203 ns p50, 527 ns p99 |
| Book-only `std::map` baseline | 208.1 ns/message |
| Book-only direct-index + SIMD | 122.7 ns/message |
| Median book-only speedup | 1.70× over seven alternating trials |
| RTL end-to-end latency at 100 MHz | 55 cycles p50, 91 p99, 173 p99.9 |
| RTL service time at 100 MHz | 19 cycles p50, 27 p99/p99.9 |
| RTL throughput from median service time | 5.26 M messages/s |
| Spartan-7 post-route | +0.058 ns WNS; 8,019 LUT, 4,523 FF, 15.5 BRAM, 6 DSP |

The book benchmark replays 500,004 decoded messages through fresh engines,
alternates execution order, reports the median of seven trials, and checks
that both engines finish with matching best prices and depths. RTL numbers are
stored as cycles and converted to nanoseconds only with the implemented
100 MHz clock; Verilator is used for cycle accuracy, not as a timing model.

## Architecture

The C++ path includes file replay plus Linux raw-socket/AF_XDP receivers, a
lock-free SPSC queue, length-checked decoding, an open-addressed order-reference
table, direct price-indexed levels, SIMD best-price scans, and market statistics.
AVX-512, AVX2, and scalar paths are selected at compile time.

The RTL path is:

```text
UDP/MoldUDP64 decapsulation → 128-bit ITCH framer → decoder
→ order-reference table → book update/best-price trackers → statistics
```

Ingest overlaps book updates. Invalid, truncated, zero-length, and oversized
frames are discarded before they can update state. The simulation model uses a
large direct-indexed book; the FPGA configuration uses a 1,024-level price band
and a smaller BRAM-backed order-reference table.

The repository also contains a queue-aware experimental trading path. It turns
completed trade bars into bytecode strategy signals and models marketable
sweeps, displayed quantity ahead, partial fills, cancels, and per-fill fees.
It remains separate from the RTL equivalence stream.

## Verification and FPGA flow

- 143,796 C++ checks, including parser boundary cases and differential tests
  against `std::unordered_map`/`std::map` references.
- RTL tests for framing, decoding, MoldUDP64/UDP stripping, hash-table behavior,
  book updates, overlapping ingest, and 20K-message differential replay.
- A board-chain test bit-bangs the framed UART-like link through the exact
  `SYNTHESIS` configuration and checks snapshots plus corruption recovery.
- `make bitstream BOARD=urbana` performs synthesis, place-and-route, timing and
  DRC checks for the Spartan-7 xc7s50 target before writing a bitstream.

The latest routed timing/utilization values are kept in
[docs/results.json](docs/results.json). Hardware has not yet been validated on
the physical board; the checked result is simulation plus Vivado implementation.
